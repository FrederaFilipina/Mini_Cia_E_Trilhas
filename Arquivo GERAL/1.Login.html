Mini Cia && Trilhas

01. Login


Objetivo:

  • Validar o acesso e permitir que o usuário acesse os recuros da plataforma
  • Alterar a senha em caso de esquecimento

Processo de
Funcionamento:

  • Login
    • Entradas:
      • Acesso as informaçõe do Cadastro do usuário

    • Execução:
      1. Preencher os campos obrigatórios
        • Login
          • Qual vai ser utilizado? (Nome / Nome de Usuário / E-mail / CPF)
        • Senha
      2. Clicar no botão de Login
        • Tem de que clica no botão ou se apertar enter no teclado vai funcionar também?
      3. Validar as informações
      4. Se informações inseridas são verdadeiras, liberar acesso ao usuário
      5. Se informações iseridas são falsas, informar erro e solicitar nova tentativa
        • Como o erro vai ser informado?
        • Como solicitar a nova tentativa?

    • Saídas:
      • Acesso aos recursos para criar, editar, visulizar e aceitar eventos
      • Acesso as informações completas sobre as trlhas
      • Acesso a avaliar a trilha e participantes do evento

    • Dúvidas:
      • Vai ter um limite de tentativas antes de bloquear o acesso ou depois de x tentativas vamos apenas solicitar que o usuário faça uma nova senha?


  • Troca de Senha (por esquecimento)
    • Entradas:
      • Acesso as informaçõe do cadastro do usuário

    • Execução:
      1. Clicar na opção de trocar de senha
        • Definir layout de funcionamento da plataforma!
      2. Digitar o e-mail ou CPF
        • Definir qual informação vai ser usado para iniciar o processo de troca de senha
      3. Definir como será as próximas etapas desse processo!
        • Ele vai receber uma senha por e-mail, ou SMS, como vai funcinar?
        • Ele vai receber um e-mail já com a solicitação do preenchimento de senha?
        • Ele vai receber um e-mail com um link para efetuar a troca da senha?

    • Saídas:
      • Troca de senha efatuada
